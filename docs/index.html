Ignition Browser

A disposable, isolated browser for macOS on Apple Silicon. Every link opens in a throwaway microVM running Firefox — fresh each time, nothing persists.

Download latest release

What it is

Ignition Browser is a menu-bar app. Pick New Ignition Browser (or send a link to it from another app) and it boots a tiny virtual machine — its own kernel, root filesystem, and network stack — restored from a warm snapshot in a fraction of a second. Firefox runs inside that VM. When you are done, the VM is discarded along with everything it touched: cookies, cache, history, downloads, any drive-by code.

  • Isolated. The page runs in a hardware-virtualized guest (Apple Hypervisor.framework), not on your Mac.
  • Disposable. The guest disk is an in-RAM overlay; it never diverges, and each session starts from the same clean snapshot.
  • User-mode networking. Traffic is NAT’d through an unprivileged user-space stack, with egress to your local network and host blocked.

Keyboard shortcuts

Host hotkeys (handled by Ignition, never passed to the page) use Ctrl+Alt. Everything else goes to Firefox.

ShortcutAction
Ctrl + Alt + RReset to a fresh disposable session (discards the current VM and relaunches from the clean snapshot)
Ctrl + Alt + SSnapshot the current VM state
Ctrl + Alt + XClose the browser window / end the session
Ctrl + LFocus the address bar (Firefox)
Ctrl + RReload page  ·  Ctrl + Shift + R hard reload (Firefox)
Ctrl + + / -Zoom in / out (Firefox)
Ctrl + FFind on page (Firefox)

Requirements

  • Apple Silicon Mac (M-series).
  • macOS 15 or later.
  • ~1 GB free disk for the guest image, built once on first launch.
